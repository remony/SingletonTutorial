Pro's and Con's of The Singleton Pattern

Pros1

  • Since the singleton is initialized when it is first accessed it doesn’t create instances of classes that are never accessed. This is good for memory management.
  • It ensures that only one instance of a class is created.
  • When no one is using it the singleton is not initialized.
  • The alternative to using singletons is static classes however there is one main issue with this: automatic initialization. The static class cannot access information which is not known until the program is running. Using lazy initialization the singletons are initialized as late as possible so it can access information when running.

Cons

  • Singletons break the single responsibility of object oriented design. A singleton class has the responsibility of its main activity and the responsibility of creating an instance of itself.2
  • It’s harder to find bugs within the program if they are globally accessed. You will have to look through a lot more code to find the problem if its accessed globally.
  • It can encourage coupling – As each classes are globally accessed it is easier for team members to code things together that shouldn’t be together. If singletons are not used then no global access would mean that this code wouldn’t work and they would have to code these two things separately.
